# AI 프로젝트 감사 명세서

> **목적**: AI가 어떤 프로젝트든 읽고 바로 점검할 수 있는 종합 체크리스트 템플릿.
> 내가 뭘 몰라도, 서비스 규모만 알면 AI가 "지금 이 프로젝트에 뭐가 없는지" 판단한다.

## 사용법

### 새 프로젝트 시작할 때
```
AUDIT_SPEC.md 파일을 읽었어.

서비스 프로필:
- 서비스 유형: REST API
- 예상 MAU: 5,000
- 팀 규모: 혼자
- 데이터 민감도: 중간 (사용자 이메일, 구매 내역)
- 업타임: 99%
- 배포 환경: Docker + 단일 서버
- 외부 의존성: PostgreSQL, Redis
- 공개 여부: 인터넷 공개

이 프로필 기반으로 이 서비스에 반드시 필요한 항목들 목록을 먼저 알려줘.
그 다음 [FastAPI / NestJS / 선택한 프레임워크]로 기능 코드 전에 골조를 잡아줘.
```

### 기존 프로젝트 점검할 때
```
AUDIT_SPEC.md 파일을 읽었어.

서비스 프로필: [위 yaml 채워서]

[코드 붙여넣기 또는 파일 목록]

각 체크리스트 항목마다 아래 형식으로 결과 줘:
✅ [항목번호] 항목명 — 확인 위치
❌ [항목번호] 항목명 — 없는 것 + 해결 코드
⚠️ [항목번호] 항목명 — 부족한 점
⏭️ [항목번호] 항목명 — 이 서비스에서 불필요한 이유

마지막에 ❌ 항목들을 우선순위순으로 정렬해서 "이것부터 고쳐라" 리스트 줘.
```

### 특정 영역만 리팩토링할 때
```
AUDIT_SPEC.md 섹션 [번호]([영역명])만 기준으로 아래 코드를 점검하고
문제 있는 부분 바로 수정 코드로 줘.

[코드]
```

## 파일 구조
```
AUDIT_SPEC.md   ← 메인 체크리스트 (이걸 AI에 던지면 됨)
README.md       ← 사용법
```
